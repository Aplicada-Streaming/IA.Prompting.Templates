# Reconfiguración de IP propia de un contenedor

> **Uso**: copiá este prompt, reemplazá los nombres de contenedor e IP por los de tu entorno y ejecutá.

---

# Contexto

El contenedor de administración de Docker (`portainer`) responde actualmente a la IP del host, en lugar de tener una IP propia.

---

# Objetivo

El contenedor `portainer` debe responder a una IP propia.

---

# Solicitudes

- Reconfigurar el contenedor `portainer` siguiendo el modelo de configuración de IP del contenedor `discord-bot-moderador`, para que responda a la IP `192.168.1.110`.
- Actualizar la documentación afectada, la base de conocimiento, la indexación y el changelog documental.

---

# Restricciones

- Antes de aplicar cambios, consultar el plan con el usuario.
- No inventar información; toda afirmación debe estar respaldada por evidencia verificable.
