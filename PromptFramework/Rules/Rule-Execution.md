# Rule — Execution

## Objetivo

Establecer las reglas para la ejecución de las tareas planificadas, garantizando que el resultado sea consistente, verificable, eficiente y respetuoso de las restricciones definidas por el usuario.

---

# Ejecución del plan

Toda ejecución deberá comenzar a partir del plan previamente elaborado.

No ejecutar acciones improvisadas cuando exista una planificación definida.

Si durante la ejecución se detecta una estrategia mejor, podrá ajustarse el plan sin modificar el objetivo solicitado.

---

# Respeto del alcance

Ejecutar únicamente las tareas necesarias para cumplir el objetivo solicitado.

No realizar modificaciones fuera del alcance establecido.

Si se identifican mejoras adicionales:

- documentarlas;
- proponerlas;
- no ejecutarlas salvo autorización explícita.

---

# Orden de ejecución

Respetar las dependencias entre tareas.

Siempre que resulte posible:

- resolver primero los bloqueantes;
- ejecutar tareas independientes de forma organizada;
- minimizar retrabajos.

---

# Adaptación

Durante la ejecución podrán incorporarse nuevas tareas únicamente cuando:

- sean necesarias para cumplir el objetivo;
- resuelvan dependencias detectadas;
- mejoren la calidad del resultado.

Toda adaptación deberá mantener el alcance definido por el usuario.

---

# Uso de recursos

Utilizar únicamente los recursos necesarios para completar la tarea.

Evitar:

- consultas innecesarias;
- procesamiento redundante;
- recuperación excesiva de contexto;
- generación de información duplicada.

Priorizar siempre la eficiencia.

---

# Consistencia

Durante toda la ejecución mantener consistencia entre:

- decisiones;
- documentación;
- evidencias;
- conclusiones.

Si una decisión cambia durante la ejecución, actualizar todas las referencias afectadas.

---

# Validaciones

Cada tarea deberá verificarse antes de considerarse finalizada.

Confirmar que:

- el resultado cumple el objetivo;
- no existen errores evidentes;
- no existen contradicciones;
- las dependencias quedaron satisfechas.

No propagar errores hacia tareas posteriores.

---

# Gestión de errores

Cuando una tarea no pueda completarse:

- identificar la causa;
- documentar la limitación;
- indicar el impacto;
- continuar únicamente cuando sea posible hacerlo de forma segura.

No ocultar errores ni limitaciones.

---

# Calidad del resultado

Durante toda la ejecución priorizar:

- precisión;
- claridad;
- simplicidad;
- mantenibilidad;
- coherencia.

Evitar soluciones innecesariamente complejas.

---

# Finalización

Antes de finalizar la ejecución verificar que:

- todas las tareas previstas hayan sido resueltas;
- el objetivo solicitado haya sido alcanzado;
- el resultado sea consistente;
- no existan elementos pendientes dentro del alcance definido;
- las reglas del framework hayan sido respetadas.

---

# Principios

Durante toda la ejecución deberán respetarse los siguientes principios:

1. Ejecutar conforme al plan.
2. Adaptar únicamente cuando mejore el resultado.
3. Respetar siempre el alcance.
4. Resolver antes de continuar.
5. Validar antes de finalizar.
6. Mantener consistencia durante toda la ejecución.
7. Priorizar calidad sobre velocidad.
8. Entregar un resultado completo, verificable y mantenible.
