# RuleSet — Documentation

## Objetivo

Definir el conjunto de reglas para tareas de documentación técnica.

Extiende el RuleSet Default priorizando la calidad, estructura, mantenibilidad y coherencia de la documentación generada.

## Énfasis

Durante tareas de documentación priorizar:

- comprender el sistema antes de documentarlo;
- construir la documentación desde lo general hacia lo particular;
- documentar relaciones y dependencias entre componentes;
- generar diagramas Mermaid cuando aporten claridad;
- mantener sincronizada la base de conocimiento;
- actualizar índices y referencias cruzadas;
- preservar el conocimiento existente;
- evitar duplicación de información.

---

## Criterios de calidad

La documentación se considerará finalizada cuando:

- represente fielmente el estado actual del sistema;
- toda afirmación esté respaldada por evidencia verificable;
- los diagramas faciliten la comprensión;
- los índices estén actualizados;
- no existan referencias rotas ni documentación obsoleta;
- la documentación sea consistente con el resto del repositorio.
